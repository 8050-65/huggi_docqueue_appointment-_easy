# Flutter Patient App MVP — Implementation Plan
**Version:** 1.0  
**Target:** iOS 12.0+ / Android API 24+ (Flutter 3.24+)  
**Architecture:** Clean Architecture + Riverpod  
**Estimated Scope:** 80–120 hours for solo developer  

---

## Project Goal

Single-screen Flutter app deployed to iOS App Store and Google Play Store. Patient logs in via Firebase phone OTP, books/checks appointments, joins queue, sees real-time position.

**MVP Scope:**
- Phone OTP login (Firebase Auth)
- View my appointments
- Join queue / view position
- Real-time position polling
- Token refresh on 401
- Logout

**Out of Scope:**
- Staff/admin dashboard (web only)
- In-app payments (link to Razorpay web)
- Video consultation
- Offline support
- Push notifications (Phase 2)

## DioClient with Token Refresh Interceptor

Location: `lib/core/network/dio_client.dart`

```dart
class DioClient {
  late Dio _dio;
  final SecureStorage _storage;
  Future<String>? _refreshPromise;

  DioClient(this._storage) {
    _dio = Dio(
      BaseOptions(
        baseUrl: AppConfig.apiBaseUrl,
        contentType: 'application/json',
        connectTimeout: const Duration(seconds: 30),
        receiveTimeout: const Duration(seconds: 30),
      ),
    );

    // Token injection interceptor
    _dio.interceptors.add(
      InterceptorsWrapper(
        onRequest: (options, handler) async {
          final token = await _storage.getAccessToken();
          if (token != null) {
            options.headers['Authorization'] = 'Bearer $token';
          }
          return handler.next(options);
        },
        onError: (error, handler) async {
          if (error.response?.statusCode == 401) {
            try {
              await _refreshAccessToken();
              // Retry original request
              return handler.resolve(
                await _dio.request(
                  error.requestOptions.path,
                  options: Options(method: error.requestOptions.method),
                ),
              );
            } catch (_) {
              // Refresh failed — logout
              await _storage.clearTokens();
              return handler.next(error);
            }
          }
          return handler.next(error);
        },
      ),
    );
  }

  Future<void> _refreshAccessToken() async {
    // Serialize concurrent 401s to avoid multiple refresh requests
    _refreshPromise ??= _doRefresh();
    try {
      await _refreshPromise;
    } finally {
      _refreshPromise = null;
    }
  }

  Future<void> _doRefresh() async {
    final refreshToken = await _storage.getRefreshToken();
    if (refreshToken == null) throw Exception('No refresh token');

    final response = await _dio.post(
      '/auth/refresh',
      data: {'refreshToken': refreshToken},
    );

    final newTokens = AuthTokensModel.fromJson(response.data);
    await _storage.saveTokens(newTokens);
  }

  // Public methods for datasources
  Future<Response> get(String path) => _dio.get(path);
  Future<Response> post(String path, {required dynamic data}) =>
      _dio.post(path, data: data);
  // ... other HTTP methods
}
```

## Authentication Flow

### Phase 1: Phone Input
1. User enters phone number (10 digits or +91 prefixed)
2. `PhoneInputScreen` calls `FirebaseAuth.signInWithPhoneNumber()`
3. Firebase sends OTP via SMS (Firebase free 10k/month)
4. UI transitions to OTP verification screen

### Phase 2: OTP Verification
1. User enters 6-digit OTP
2. `OtpVerificationScreen` calls Firebase to verify
3. Firebase returns `UserCredential` with ID token
4. Call `POST /api/auth/patient/login { idToken }`
5. NestJS backend:
   - Verifies ID token with Firebase Admin SDK
   - Looks up patient by phone
   - Issues Huggi JWT pair
6. Save tokens to `FlutterSecureStorage`
7. Transition to authenticated state

### Phase 3: Authenticated Session
1. `DioClient` injects `Authorization: Bearer <accessToken>` on all requests
2. On 401 response, trigger refresh:
   - `POST /auth/refresh { refreshToken }`
   - Save new tokens
   - Retry original request
3. On refresh failure (401 again), logout and navigate to login

### Phase 4: Logout
1. User taps logout button
2. Call `POST /auth/logout { refreshToken }`
3. Clear tokens from secure storage
4. Navigate to login screen

## Routing with GoRouter

Location: `lib/config/router.dart`

```dart
final goRouterProvider = Provider((ref) {
  final authState = ref.watch(authNotifierProvider);

  return GoRouter(
    initialLocation: authState.when(
      authenticated: (_) => '/home',
      unauthenticated: () => '/login',
      loading: () => '/loading',
      error: (_) => '/login',
    ),
    routes: [
      GoRoute(
        path: '/login',
        builder: (context, state) => PhoneInputScreen(),
        routes: [
          GoRoute(
            path: 'otp',
            builder: (context, state) => OtpVerificationScreen(
              phoneNumber: state.extra as String,
            ),
          ),
        ],
      ),
      GoRoute(
        path: '/home',
        builder: (context, state) => HomeScreen(),
        routes: [
          GoRoute(
            path: 'appointments',
            builder: (context, state) => AppointmentsListScreen(),
          ),
          GoRoute(
            path: 'queue',
            builder: (context, state) => QueueStatusScreen(),
          ),
          GoRoute(
            path: 'profile',
            builder: (context, state) => ProfileScreen(),
          ),
        ],
      ),
      GoRoute(
        path: '/loading',
        builder: (context, state) => LoadingScreen(),
      ),
    ],
    redirect: (context, state) {
      final isAuthenticated = ref.read(isLoggedInProvider);
      if (!isAuthenticated && state.location != '/login') {
        return '/login';
      }
      if (isAuthenticated && state.location == '/login') {
        return '/home';
      }
      return null;
    },
  );
});
```

## Repository Pattern

### Appointment Repository Interface

```dart
abstract class AppointmentRepository {
  Future<List<Appointment>> getMyAppointments();
}

class AppointmentRepositoryImpl implements AppointmentRepository {
  final AppointmentRemoteDataSource _remote;

  AppointmentRepositoryImpl(this._remote);

  @override
  Future<List<Appointment>> getMyAppointments() async {
    try {
      final models = await _remote.getMyAppointments();
      return models.map((m) => Appointment(
        id: m.id,
        appointmentTime: m.appointmentTime,
        status: m.status,
        doctor: Doctor(
          id: m.doctor.id,
          name: m.doctor.user.name,
          specialization: m.doctor.specialization,
        ),
      )).toList();
    } on DioException catch (e) {
      throw ApiException.fromDioException(e);
    }
  }
}
```

```dart
class AppointmentRemoteDataSource {
```

## Error Handling

### ApiException

```dart
class ApiException implements Exception {
  final String message;
  final int? statusCode;
  final dynamic originalError;

  ApiException({
    required this.message,
    this.statusCode,
    this.originalError,
  });

  factory ApiException.fromDioException(DioException e) {
    final statusCode = e.response?.statusCode;
    final message = switch (statusCode) {
      401 => 'Your session expired. Please log in again.',
      403 => 'You do not have access to this resource.',
      404 => 'Resource not found.',
      409 => 'This action is not allowed (conflict).',
      >= 500 => 'Server error. Please try again later.',
      _ => e.message ?? 'An error occurred',
    };
    return ApiException(
      message: message,
      statusCode: statusCode,
      originalError: e,
    );
  }
}
```

### AuthStateNotifier (with error handling)

```dart
class AuthNotifier extends StateNotifier<AuthState> {
  final AuthRepository _auth;

  AuthNotifier(this._auth) : super(const AuthState.unauthenticated());

  Future<void> patientLogin(String idToken) async {
    state = const AuthState.loading();
    try {
      final tokens = await _auth.patientLogin(idToken);
      state = AuthState.authenticated(tokens);
    } on ApiException catch (e) {
      state = AuthState.error(e.message);
    }
  }
}
```

## Key Technical Decisions

| Decision | Reason |
|---|---|
| **Riverpod** over GetX | Easier to test, functional, no BuildContext needed |
| **GoRouter** over Beamer | Built-in deeplink support, official maintenance |
| **Dio** over http | Interceptor support, request/response transforming |
| **FlutterSecureStorage** | Encrypted on iOS (Keychain), Android (Keystore) |
| **Firebase Auth** (client) | Free OTP, native Firebase Admin verification on backend |
| **No offline-first** | MVP users expect real-time queue position; offline complexity not worth it |
| **No BLoC** | Riverpod simpler for small app; avoid provider boilerplate |
